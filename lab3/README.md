## Формулировка задания

Упростить выражения, убрав из него все произведения, в которых в качестве сомножителя используется нуль.

## Структура проекта

1. `structures.h`, `structures.c` — структура узла дерева и создание узлов.
2. `stack.h`, `stack.c` — простые стеки для символов и узлов.
3. `parse.h`, `parse.c` — перевод выражения в ОПН и общий разбор выражения.
4. `tree.h`, `tree.c` — построение дерева по ОПН, печать, вычисление и упрощение.
5. `main.c` — консольная программа для ввода выражений и вывода ОПН.
6. `tests.c` — тесты на ОПН, дерево и преобразование.

## Как запустить

Запуск основной программы:

```bash
make main
```

Запуск тестов:

```bash
make test_all
```

## Ключевая идея

Пайплайн программы такой:
- ввод выражения в обычной инфиксной форме;
- перевод выражения в ОПН в модуле `parse` с помощью модуля `stack`;
- построение дерева выражения по полученной ОПН в модуле `tree`;
- рекурсивное упрощение дерева.

Для унарного минуса в ОПН используется отдельный оператор `~`.
Например, `2*-3` переводится в `2 3 ~ *`.

После построения дерева выполняется преобразование варианта 13:
если у узла `*` хотя бы один потомок равен `0`, всё это произведение заменяется числом `0`.

## Что проверяют тесты

- корректный перевод в ОПН с учётом приоритетов;
- корректный перевод в ОПН для унарного минуса;
- построение дерева из ОПН;
- выражение с одним произведением на ноль;
- выражение с несколькими произведениями на ноль в разных местах дерева;
- корректный разбор унарного минуса;
- случай `-x*0`, который после упрощения должен давать `0`.
